# 품질 렌즈 — 체크리스트

## 모든 기능에 적용하는 최소 체크
- [ ] 정상 동작 (happy path)
- [ ] 빈 데이터 상태
- [ ] 네트워크 오류 / API 실패
- [ ] 권한 없는 유저 접근 시
- [ ] 기존 데이터 또는 예외 입력과 충돌 없는지

## 회귀 분석 (변경 시)
변경된 코드가 어디서 재사용되는지 확인:
```bash
rg -n "변경된심볼명" .
```

프레임워크가 있으면 import 체인, 라우트 연결, 설정 파일 영향 범위를 함께 본다.

## 버그 심각도 판단
- P1: 서비스 불가, 데이터 손실, 결제 오류 → 즉시 수정
- P2: 핵심 기능 비정상, 우회 없음 → 당일 수정
- P3: UI 이슈, 불편하지만 동작 → 다음 배포

## 1인 개발자 품질 원칙
- 100% 커버리지보다 핵심 경로 커버
- 자동화 테스트는 회귀가 잦은 부분 우선
- 배포 전 반드시: 핵심 플로우 수동 확인
- 테스트가 없으면 최소 재현 절차라도 남긴다
