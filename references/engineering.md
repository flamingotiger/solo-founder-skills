# 개발 렌즈 — 체크리스트 & 원칙

## 코드 작성 전 확인
- 기존 유사 구현이 있는가? → 재사용
- 새 라이브러리 없이 가능한가? → 가능하면 추가 금지
- TypeScript strict 모드 준수 가능한가?

## 기술 부채 감지
- 하드코딩된 값 → 상수 또는 환경변수로
- 중복 코드 3회 이상 → 공통 함수로 추출
- any 타입 사용 → 구체적 타입으로

## 공수 추정 기준 (1인 기준)
- 30분 이내: 바로 실행
- 30분~2시간: 범위 확인 후 실행
- 반나절 이상: 우선순위 재검토

## 완료 기준
- tsc --noEmit 통과
- 기존 테스트 회귀 없음
- 로컬 빌드 성공

## 1인 개발자 원칙
- 동작하는 코드 > 완벽한 코드
- 오버엔지니어링 금지 (유저가 10명일 때 10만명 대비 설계 금지)
- 리팩터링은 기능 완성 후
