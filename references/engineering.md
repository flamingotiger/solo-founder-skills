# 개발 렌즈 — 체크리스트 & 원칙

## 코드 작성 전 확인
- 기존 유사 구현이 있는가? → 재사용
- 새 라이브러리 없이 가능한가? → 가능하면 추가 금지
- 현재 프로젝트의 lint/type/test 규칙을 지킬 수 있는가?
- 배포 환경이나 런타임 제약과 충돌하지 않는가?

## 기술 부채 감지
- 하드코딩된 값 → 상수 또는 환경변수로
- 중복 코드 3회 이상 → 공통 함수로 추출
- 임시 분기나 예외 처리 → 제거 조건을 남김
- 타입 시스템이 있으면 느슨한 타입보다 구체화 우선

## 공수 추정 기준 (1인 기준)
- 30분 이내: 바로 실행
- 30분~2시간: 범위 확인 후 실행
- 반나절 이상: 우선순위 재검토

## 완료 기준
- 프로젝트가 가진 기본 검증 통과
- 기존 핵심 흐름 회귀 없음
- 변경 범위에 맞는 최소 검증 완료
- 배포 시 운영 리스크를 설명할 수 있음

## 1인 개발자 원칙
- 동작하는 코드 > 완벽한 코드
- 오버엔지니어링 금지 (유저가 10명일 때 10만명 대비 설계 금지)
- 리팩터링은 기능 완성 후
- 스택 교체보다 현재 코드베이스에 맞는 수정 우선
