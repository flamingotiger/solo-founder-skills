# 디자인 렌즈 — 체크리스트 & 패턴

## 일관성 체크
- 기존 컴포넌트 또는 UI 패턴 재사용 가능한가?
- 색상, 간격, 타이포그래피는 프로젝트 토큰/스타일 시스템을 우선하는가?
- 새 패턴을 넣는다면 기존 화면과 충돌하지 않는가?

## 직관성 체크 (설명 없이 쓸 수 있는가)
- 버튼 라벨이 행동을 명확히 설명하는가?
- 오류 메시지가 해결 방법을 포함하는가?
- 빈 상태(empty state)가 다음 행동을 안내하는가?

## 반응형 체크
- 모바일(375px): 터치 타겟 최소 44px
- 태블릿(768px): 레이아웃 변화 확인
- 데스크탑(1280px): 기본 설계 기준
- 화면 크기와 입력 방식(마우스/터치/키보드)에 따라 사용성이 유지되는가?

## 1인 개발자 디자인 원칙
- 완벽한 디자인보다 일관된 디자인
- 새 패턴 도입 전 기존 패턴 재사용 시도
- 애니메이션은 기능 완성 후에
- 접근성은 별도 기능이 아니라 기본 사용성으로 본다
