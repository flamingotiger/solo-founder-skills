# 디자인 렌즈

## 판단 기준
"예쁜가"가 아니라 "이해되는가, 믿을 수 있는가, 행동하게 만드는가"로 본다.
목표 톤: IBM/WordPress 계열 — 정보 밀도 있고, 구조 명확하고, 스타일보다 시스템이 먼저.

## AI 생성 티 신호 (발견 즉시 지적)
- radius 과대 → 장난감처럼 보임
- 이모지로 위계 대체
- 강한 그림자/그라디언트 → 구조보다 효과가 먼저 보임
- 카드형 박스 남발 → 화면이 조각남
- 긴 설명 텍스트 → 인터페이스가 아닌 문서처럼 보임

## 시각 디폴트

| 요소 | 기준 |
|------|------|
| radius | 모서리 부분 radius를 살짝 준다. radius가 과함 |
| shadow | 얕게, 구조 보조용. 다층 금지 |
| gradient | 브랜딩 구간에만. 배경/카드 반복 금지 |
| color | neutral + key color 1개. key color는 CTA/활성 상태에만 |
| card | 독립된 묶음에만. 리스트/폼/설정은 카드 없이 |
| text | 제목+보조문구+CTA만으로. 설명 문단 최소화 |

## 우선순위
1. **이해** — 3초 안에 화면 목적과 해야 할 행동이 하나로 보이는가
2. **신뢰** — 간격/정렬/상태 표현이 안정적인가. 에러·로딩·빈 상태 있는가
3. **속도** — 읽는 순서 자연스러운가. 불필요한 모달/카드/설명 없는가

## 필수 상태 체크
`default / hover / focus / active / disabled / loading / success / error / empty`
빠진 상태가 있으면 구현 완료 아님.

## 민감 액션
삭제·결제·탈퇴는 일반 액션과 시각적으로 구분. irreversible이면 결과 미리 설명.

## 폼 UX
- placeholder는 예시, 라벨 대체 불가
- 에러는 제출 후가 아니라 가능한 빨리
- 필수/선택 구분 명확히

## 바로 지적할 허점
- CTA보다 보조 카드가 더 강하게 보임
- 타이포 단계가 없어 읽는 순서가 없음
- radius·shadow·gradient는 강한데 구조는 약함
- 빈 상태에 다음 행동이 없음
- key color가 배경/카드/텍스트에 동시에 퍼짐
